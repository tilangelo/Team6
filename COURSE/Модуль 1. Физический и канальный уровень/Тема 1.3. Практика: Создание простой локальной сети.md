# Тема 1.3. Практика: Создание простой локальной сети

**Цель работы:** Научиться физически и логически объединять несколько компьютеров в единую локальную сеть (LAN) для обеспечения обмена данными и доступа к общим ресурсам.

## 1. Теоретическая база

Локальная сеть (LAN) — это группа устройств (ПК, ноутбуков, принтеров), соединенных между собой на небольшой территории (офис, дом, этаж здания). В простейшей сети используется топология «Звезда»: все устройства подключаются к одному центральному устройству — коммутатору (свитчу).

**Необходимые компоненты:**
*   **Компьютеры:** 2 и более с сетевыми адаптерами (Ethernet).
*   **Коммутатор (Switch):** Центральное устройство (на 4/8 портов).
*   **Патч-корды (кабель UTP cat5e/cat6):** Для подключения ПК к свитчу.
*   **IP-адреса:** Уникальные идентификаторы устройств в сети (например, `192.168.1.1`).

## 2. Схема простой сети

В качестве примера создадим сеть из 3 компьютеров:

```text
       [Компьютер 1]          [Компьютер 2]          [Компьютер 3]
             |                        |                        |
         (патч-корд)              (патч-корд)              (патч-корд)
             |                        |                        |
             +------------+-----------+------------+
                          |
                   [Коммутатор (Switch)]
                          |
                          +--- (опционально: выход в Интернет)
```

## 3. Пошаговая инструкция по созданию сети

### Этап A: Физическое подключение (Hardware)

1.  **Подготовка:** Убедитесь, что все компьютеры выключены. Распакуйте коммутатор и подключите его к розетке 220В.
2.  **Соединение:**
    *   Возьмите патч-корд (кабель с коннекторами RJ-45).
    *   Один конец вставьте в сетевую карту компьютера (сзади системного блока, разъем похож на широкий телефонный).
    *   Другой конец вставьте в любой свободный порт коммутатора (обычно они пронумерованы 1, 2, 3...).
3.  **Повтор:** Выполните шаг 2 для всех компьютеров.
4.  **Индикация:** На коммутаторе и на сетевых картах ПК должны загореться зеленые/оранжевые светодиоды (Link/Activity). Если не горят — проверьте надежность защелки коннектора.

### Этап B: Программная настройка (Software) — Windows 10/11

После физического подключения компьютеры увидят сеть, но не смогут общаться, так как у них нет правильных адресов. Назначим их вручную.

**На каждом компьютере выполните:**

1.  Откройте **Панель управления** → **Центр управления сетями и общим доступом**.
2.  Слева нажмите **«Изменение параметров адаптера»**.
3.  Найдите **«Ethernet»** (или «Подключение по локальной сети»), нажмите правой кнопкой → **«Свойства»**.
4.  В списке компонентов найдите **«IP версии 4 (TCP/IPv4)»** → нажмите **«Свойства»**.
5.  Выберите переключатель: **«Использовать следующий IP-адрес»**.
6.  Заполните поля (пример для 3 ПК):

| Устройство     | IP-адрес       | Маска подсети   | DNS-сервер (пока не нужен) |
|----------------|----------------|-----------------|------------------------------|
| **Компьютер 1** | 192.168.1.10   | 255.255.255.0   | (оставить пустым или 8.8.8.8) |
| **Компьютер 2** | 192.168.1.20   | 255.255.255.0   | (оставить пустым)            |
| **Компьютер 3** | 192.168.1.30   | 255.255.255.0   | (оставить пустым)            |

**Правило:** Первые три числа (192.168.1) — это *сеть*, они должны быть **одинаковыми** у всех. Последнее число — *узел*, он должен быть **уникальным** в пределах этой сети.

7.  Нажмите «OK» → «Закрыть».

### Этап C: Проверка связи (Тестирование)

Воспользуемся утилитой `ping`. Она отправляет эхо-запрос и ждет ответ.

1.  На **Компьютере 1** нажмите `Win + R`, введите `cmd` и нажмите Enter.
2.  В черном окне (командной строке) введите:
    ```cmd
    ping 192.168.1.20
    ```
3.  Если настройка верна, вы увидите ответы:
    ```text
    Ответ от 192.168.1.20: число байт=32 время<1мс TTL=128
    ```
    *(Ошибка «Превышен интервал ожидания» означает, что связи нет).*

4.  Повторите `ping` для Компьютера 3.
5.  Попробуйте пропинговать компьютер сам себя:
    ```cmd
    ping 127.0.0.1
    ```
    *(Этот адрес «localhost» — проверяет, работает ли ваш сетевой стек Windows).*

## 4. Настройка общего доступа к папкам (File Sharing)

Чтобы обмениваться файлами, нужно открыть общий доступ.

1.  На любом компьютере создайте папку, например `C:\Общая_папка`.
2.  Нажмите на папку правой кнопкой → **Свойства** → вкладка **«Доступ»**.
3.  Нажмите кнопку **«Общий доступ...»**.
4.  В выпадающем списке выберите **«Все»** (или «Everyone»), нажмите «Добавить».
5.  Установите уровень разрешений: «Чтение» (только смотреть) или «Чтение и запись» (можно копировать и удалять).
6.  Нажмите «Общий доступ» → «Готово».

**Как открыть эту папку с другого ПК:**
*   Откройте «Проводник» (Win+E).
*   В адресной строке введите: `\\192.168.1.10` (IP того компьютера, где лежит папка).
*   Вы увидите расшаренные ресурсы.

## 5. Типичные ошибки и их решение

| Симптом | Вероятная причина | Решение |
| :--- | :--- | :--- |
| **Светодиод на порту не горит** | Обрыв кабеля или не защелкнут коннектор | Переобжать кабель, проверить подключение |
| **Команда ping пишет «Сбой сети»** | Неправильная маска (255.255.255.0) или IP вручную не назначен | Проверить настройки IPv4 на всех ПК |
| **Пинг проходит, но нет доступа к папкам** | Отключено сетевое обнаружение или общий доступ к файлам | В «Центре управления сетями» → «Дополнительные параметры» → Включить сетевое обнаружение и общий доступ (для частной сети) |
| **Windows просит пароль при доступе** | Сработала проверка учетных записей | Добавить гостя: Свойства папки → Доступ → Добавить пользователя «Guest» |

## 6. Итог практической работы

* Мы создали физическую топологию «Звезда» на основе коммутатора.  
* Назначили статические IP-адреса из диапазона частных адресов (RFC 1918).  
* Проверили связность с помощью утилиты `ping`.  
* Настроили простейший файловый сервер на базе общей папки Windows.

**Задание для самостоятельной работы:**
Проведите мониторинг трафика (попробуйте скопировать фильм с ПК1 на ПК2 и обратно, замерив скорость в диспетчере задач → «Производительность» → «Ethernet»).
